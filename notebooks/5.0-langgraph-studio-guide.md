# LangGraph Studio Setup Guide

A step-by-step guide for running and debugging LangGraph agents visually using LangGraph Studio.

## Prerequisites

- **LangSmith account** — sign up at [smith.langchain.com](https://smith.langchain.com)
- **Docker Desktop** or **OrbStack** (v2.22.0+) — only needed for `langgraph up` (production-like testing)
- **Python ≥ 3.11**
- **API keys**: `OPENAI_API_KEY` (and optionally `TAVILY_API_KEY`)

## Quick Start (5 Steps)

### Step 1: Install the LangGraph CLI

```bash
pip install --upgrade "langgraph-cli[inmem]"
```

### Step 2: Understand the Project Structure

A LangGraph Studio project needs 3 things:

```
your-project/
├── langgraph.json    ← graph configuration
├── your_graph.py     ← Python file with a compiled graph
├── .env              ← API keys
└── requirements.txt  ← dependencies
```

**This repo includes a ready-to-use example** in `notebooks/langgraph-studio/`:

```
notebooks/langgraph-studio/
├── langgraph.json
├── simple-graph.py
├── coffee_order_graph.py
├── .env
└── requirements.txt
```

### Step 3: Configure `langgraph.json`

This file tells Studio where your graphs are. Here's the example from this repo:

```json
{
  "graphs": {
    "simple_graph": "./simple-graph.py:graph",
    "coffee_order_graph": "./coffee_order_graph.py:graph"
  },
  "env": "./.env",
  "python_version": "3.11",
  "dependencies": ["."]
}
```

**Key fields:**
- `graphs` — maps graph names to `file_path:variable_name` (the variable must be a compiled `StateGraph`)
- `env` — path to your `.env` file
- `dependencies` — list of directories to install (uses `requirements.txt` in each)

### Step 4: Set Up Your `.env`

Create a `.env` file with your API keys:

```bash
OPENAI_API_KEY=sk-...
TAVILY_API_KEY=tvly-...
```

> **Important**: Do NOT add `LANGSMITH_API_KEY` to `.env` — Studio handles LangSmith authentication automatically.

### Step 5: Launch

Navigate to your project directory and run:

```bash
cd notebooks/langgraph-studio
langgraph dev
```

This starts a local server at `http://127.0.0.1:2024`.

Then open Studio in your browser:

```
https://smith.langchain.com/studio/?baseUrl=http://127.0.0.1:2024
```

> **Safari users**: Safari blocks localhost HTTP. Use `langgraph dev --tunnel` to get an HTTPS URL instead.

## Using Studio

1. **Select a graph** from the dropdown (e.g., `simple_graph` or `coffee_order_graph`)
2. **Enter an input state** — the JSON input for your graph (e.g., `{"graph_state": "2"}`)
3. **Click Run** and watch the graph execute step by step
4. **Inspect state** at each node — see how the state changes as it flows through

Studio features:
- Visual graph execution with real-time state inspection
- Step-through debugging
- Conversation history for message-based graphs
- Hot-reload when you edit your Python files

## Alternative: Mac Desktop App

1. Download the `.dmg` from [LangGraph Studio releases](https://github.com/langchain-ai/langgraph-studio/releases)
2. Install and launch
3. Authenticate with LangSmith when prompted
4. Open your project directory (must contain `langgraph.json`)

## `langgraph dev` vs `langgraph up`

| Command | Best for | Docker needed? |
|---------|----------|----------------|
| `langgraph dev` | Development — lightweight, hot-reload | No |
| `langgraph up` | Production-like testing — full Docker Compose stack (API + Redis + PostgreSQL) | Yes |

For this course, `langgraph dev` is all you need.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| "Cannot connect to server" | Make sure `langgraph dev` is running in another terminal |
| Auth errors | Log into [smith.langchain.com](https://smith.langchain.com) in your browser first |
| Safari won't connect | Use `langgraph dev --tunnel` for HTTPS |
| Module not found | Check that `requirements.txt` includes all dependencies |
| Graph not showing | Verify `langgraph.json` points to the correct `file:variable` |
