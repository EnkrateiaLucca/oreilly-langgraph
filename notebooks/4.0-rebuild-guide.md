# Rebuild Guide: LangGraph Research Report Agent (Intro Version)

> **Purpose**: Step-by-step instructions to live-code the simplified research report pipeline during the O'Reilly intro training. Designed to be teachable in **30–45 minutes**.
>
> **Final product**: A 4-node LangGraph pipeline that takes any topic and generates a styled HTML research report with cited sources.

---

## Architecture Overview

```
START → planner → researcher → writer → compiler → END
```

**4 nodes, 4 core concepts**:

| Node | LangGraph Concept | Uses LLM? |
|------|-------------------|-----------|
| `planner` | Structured output (`with_structured_output`) | Yes |
| `researcher` | Tool integration (Tavily) + state reducer (`operator.add`) | No |
| `writer` | LLM call per section, accumulates via reducer | Yes |
| `compiler` | Pure Python node (Jinja2 → HTML) | No |

**Design principle**: LLMs decide *what* (plan, prose). Plain Python handles *how* (search, HTML assembly).

> **What changed from the previous version?** The earlier draft had 7 nodes, conditional loops, a self-review quality routing function, matplotlib chart generation, Playwright screenshots, and **5+ intermediate graphs** before the final one. That was overwhelming for an intro course. The new version keeps the pedagogical core (TypedDict + reducers + structured output + nodes/edges) and ships **one** intermediate graph (planner-only) before the final 4-node pipeline. The old notebook is preserved at `notebooks/extra-notebooks/4.0-research-report-agent.ipynb`.

## Phase 1: State Design (5–8 min)

**Teaching beat**: Explain that state is a `TypedDict` shared across nodes. By default, returning a key **overwrites** it. For lists we usually want to **accumulate** — that's a reducer.

```python
import operator
from typing import Annotated, List
from typing_extensions import TypedDict
from pydantic import BaseModel, Field

class Section(BaseModel):
    title: str = Field(description="Short, catchy section title")
    query: str = Field(description="A web search query that gathers info for this section")

class ReportPlan(BaseModel):
    sections: List[Section] = Field(description="3 to 4 sections that cover the topic")

class ReportState(TypedDict):
    topic: str
    plan: ReportPlan
    research: Annotated[list, operator.add]
    written_sections: Annotated[list, operator.add]
    final_html: str
```

**Talking points**:
- `topic` and `final_html` are plain — they get overwritten (which is fine).
- `research` and `written_sections` use `operator.add` so each node can append.

## Phase 2: Planner Node + Intermediate Graph (8–10 min)

This is the **only** intermediate graph in the lesson. Use it to demonstrate `with_structured_output` and the basic build/compile/visualize/invoke loop.

```python
from langchain.chat_models import init_chat_model

llm = init_chat_model("openai:gpt-5.4-mini")

PLANNER_PROMPT = """You are a research planner. Given a topic, produce a plan with 3-4 sections.
Each section needs a short catchy title and ONE concrete web search query."""

def planner(state: ReportState) -> dict:
    structured = llm.with_structured_output(ReportPlan)
    plan = structured.invoke([
        {"role": "system", "content": PLANNER_PROMPT},
        {"role": "user", "content": f"Topic: {state['topic']}"},
    ])
    return {"plan": plan}
```

Build, visualize, run:

```python
from langgraph.graph import StateGraph, START, END
from IPython.display import Image, display

builder = StateGraph(ReportState)
builder.add_node("planner", planner)
builder.add_edge(START, "planner")
builder.add_edge("planner", END)
mini_graph = builder.compile()

display(Image(mini_graph.get_graph().draw_mermaid_png()))

DEMO_TOPIC = "The rise of AI coding assistants in 2026"
result = mini_graph.invoke({"topic": DEMO_TOPIC, "research": [], "written_sections": [], "final_html": ""})
for s in result["plan"].sections:
    print("•", s.title, "→", s.query)
```

**Teaching beat**: "We just built a LangGraph application with one node. Everything else from here is just adding more nodes to this same pattern."

## Phase 3: Researcher Node (5 min)

```python
from tavily import TavilyClient
tavily = TavilyClient(api_key=os.environ["TAVILY_API_KEY"])

def researcher(state: ReportState) -> dict:
    findings = []
    for section in state["plan"].sections:
        resp = tavily.search(query=section.query, max_results=3)
        findings.append({
            "section": section.title,
            "query": section.query,
            "results": resp.get("results", []),
        })
    return {"research": findings}
```

**Talking point**: Returning `{"research": findings}` triggers the `operator.add` reducer.

## Phase 4: Writer Node (5 min)

```python
WRITER_PROMPT = """You are a sharp tech writer. Write ONE engaging paragraph (4-6 sentences)
for the section below using ONLY the provided research snippets. Be concrete and specific."""

def writer(state: ReportState) -> dict:
    written = []
    for finding in state["research"]:
        snippets = "\n\n".join(
            f"- {r.get('title','')}: {r.get('content','')}" for r in finding["results"]
        )
        msg = llm.invoke([
            {"role": "system", "content": WRITER_PROMPT},
            {"role": "user", "content": f"Section: {finding['section']}\n\nResearch:\n{snippets}"},
        ])
        written.append({"title": finding["section"], "body": msg.content, "sources": finding["results"]})
    return {"written_sections": written}
```

## Phase 5: Compiler Node — Pure Python (5 min)

Use this to make the point that **a node is just a Python function** — no LLM required.

```python
import jinja2, datetime

EMOJIS = ["🚀", "🔬", "💡", "📊", "🌟", "🧠"]

TEMPLATE = """<!doctype html>
<html><head><meta charset='utf-8'><title>{{ topic }}</title>
<style>
  body { font-family: -apple-system, system-ui, sans-serif; max-width: 760px; margin: 2em auto; padding: 0 1em; color: #222; line-height: 1.6; }
  h1 { font-size: 2em; border-bottom: 3px solid #3498db; padding-bottom: .3em; }
  h2 { color: #2c3e50; margin-top: 1.8em; }
  .meta { color: #888; font-size: .9em; }
  .sources { font-size: .85em; background: #f6f8fa; padding: .8em 1em; border-left: 3px solid #3498db; margin-top: .5em; }
  .sources a { color: #3498db; text-decoration: none; }
</style></head><body>
<h1>{{ topic }}</h1>
<div class='meta'>Generated {{ date }}</div>
{% for s in sections %}
  <h2>{{ s.emoji }} {{ s.title }}</h2>
  <p>{{ s.body }}</p>
  <div class='sources'><strong>Sources:</strong><ul>
    {% for r in s.sources %}<li><a href='{{ r.url }}'>{{ r.title }}</a></li>{% endfor %}
  </ul></div>
{% endfor %}
</body></html>"""

def compiler(state: ReportState) -> dict:
    sections = [
        {**s, "emoji": EMOJIS[i % len(EMOJIS)]}
        for i, s in enumerate(state["written_sections"])
    ]
    html = jinja2.Template(TEMPLATE).render(
        topic=state["topic"],
        date=datetime.date.today().isoformat(),
        sections=sections,
    )
    return {"final_html": html}
```

## Phase 6: The Final Graph (5 min)

```python
builder = StateGraph(ReportState)
builder.add_node("planner", planner)
builder.add_node("researcher", researcher)
builder.add_node("writer", writer)
builder.add_node("compiler", compiler)

builder.add_edge(START, "planner")
builder.add_edge("planner", "researcher")
builder.add_edge("researcher", "writer")
builder.add_edge("writer", "compiler")
builder.add_edge("compiler", END)

graph = builder.compile()
display(Image(graph.get_graph().draw_mermaid_png()))
```

Run it:

```python
from IPython.display import HTML

result = graph.invoke({
    "topic": DEMO_TOPIC,
    "research": [],
    "written_sections": [],
    "final_html": "",
})
HTML(result["final_html"])
```

## Wrap-up

Recap the four ideas students just used:

1. **State** with `TypedDict` + reducers (`operator.add`)
2. **Structured output** with Pydantic (`with_structured_output`)
3. **Nodes** as plain Python functions returning state updates
4. **Edges** wiring nodes into a pipeline (`add_edge` / `START` / `END`)

Suggested extensions for curious students (do not live-code):
- Add a quality-review loop with a conditional edge.
- Parallelize section writing with a fan-out pattern.
- Add a chart-generation node.
